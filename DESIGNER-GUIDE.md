# Основа — Гайд для дизайнера

Как работает плагин, что он анализирует, как подготовить макет для идеальной выгрузки и как читать результат.

## Что делает плагин

Ты выделяешь элемент в Figma → плагин анализирует его и все вложенные элементы → генерирует текстовый промпт → ты копируешь его и вставляешь в AI (Cursor, Claude) → AI генерирует код, который максимально точно повторяет твой макет.

Плагин **не генерирует код сам**. Он создаёт подробное описание элемента на языке, который AI понимает лучше всего: CSS-свойства + Tailwind-классы + структура вложенности.

## Установка

1. Скачай/склонируй проект
2. В терминале: `npm install && npm run build`
3. В Figma: Plugins → Development → Import plugin from manifest
4. Укажи путь к файлу `manifest.json` в папке проекта

## Как пользоваться

1. Открой плагин в Figma
2. Выдели **один** элемент (фрейм, компонент, текст, фигуру)
3. Плагин мгновенно покажет анализ
4. Нажми **«Скопировать промпт»** — текст попадёт в буфер
5. Вставь в AI-ассистент с инструкцией вроде «Сверстай этот элемент»

## Что анализирует плагин

### Для каждого элемента

| Свойство | Что извлекается | Пример в промпте |
|----------|-----------------|------------------|
| **Layout** | Flex/Grid направление, wrap, gap | `Auto Layout: horizontal, gap 24` или `CSS Grid: 2 cols × 2 rows; grid-template-columns: 1fr 3fr` |
| **Размер** | fill/hug/fixed + пиксели | `width: fill, height: hug` |
| **Padding** | Внутренние отступы | `padding 16 24` |
| **Фон** | Цвет, градиент, изображение | `fill #3B82F6` |
| **Скругление** | Все 4 угла | `radius 24px` |
| **Обводка** | Цвет + толщина | `stroke #E5E7EB 1px` |
| **Типографика** | Шрифт, размер, вес, интерлиньяж, цвет | `Jost 18 Regular, color #000000` |
| **Позиция** | Absolute: offsets от родителя | `absolute; top: 20px; left: 40px` |
| **Grid-позиция** | Ячейка + размер трека | `grid-area: 1 / 2; column track: 3fr` |
| **Overflow** | Обрезка контента | `clip content (overflow: hidden)` |
| **Tailwind** | Готовые CSS-классы | `flex-col items-center justify-end bg-gray-100 rounded-3xl` |

### Пустые Grid-ячейки

Если в CSS Grid есть ячейки без контента, плагин автоматически добавляет `[EMPTY CELL]` с указанием позиции (`grid-area: 1 / 3`). Это важно — AI видит полную структуру грида и не размещает контент в пустые ячейки.

### Иерархия (позиционирование)

Плагин показывает цепочку от корневого фрейма до выбранного элемента. Каждый уровень — с Tailwind-классами контейнера:

```
.page-wrapper (flex-col, gap-6, p-8)
  └─ .content-area (flex-row, gap-4, items-start)
    └─ Section [ЭТОТ ЭЛЕМЕНТ] (w-full, h-auto)
```

AI видит контекст: в каком контейнере живёт элемент, какие у родителя gap/padding/align.

### Вложенные элементы

Рекурсивный обход всех потомков (до 10 уровней глубины):

```
|- Card -- Auto Layout: vertical; grid-area: 2 / 2; column track: 3fr; width: 760, height: 460; fill #f0f2f3; radius 24px [flex-col bg-gray-100 rounded-3xl]
  |- image 32 -- width: 410, height: 553; fill #ff0000 [bg-[#ff0000]]
  |- Круг -- absolute; top: -40px; left: -40px; width: 40, height: 40; fill #d9d9d9; radius 9999px [bg-zinc-300 rounded-full]
|- [EMPTY CELL] -- grid-area: 1 / 2
```

### Parent Context (опционально)

Тоггл «Include parent context» добавляет подробности о непосредственном родителе: layout, gap, align, padding, позиция ребёнка в списке. Полезно когда AI должен понимать, как элемент сидит в контексте.

## Как подготовить макет для идеальной выгрузки

### Auto Layout — обязательно

Плагин предупредит, если на фрейме нет Auto Layout. Без него дети будут описаны как `absolute`, и AI получит пиксельные координаты вместо flex/grid — код будет хрупким.

**Правило:** каждый фрейм с детьми должен иметь Auto Layout (Shift+A).

### CSS Grid в Figma

Figma поддерживает CSS Grid layout. Плагин умеет:
- Читать количество колонок/рядов
- Читать размеры треков (fr, px, auto)
- Читать позицию каждого ребёнка (row, col, span)
- **Автоматически вычислять пропорции** для auto-треков
- **Распознавать пустые ячейки** и добавлять placeholder `[EMPTY CELL]`

Пример: если у тебя Grid 2×2 с колонками `auto auto`, а фактически в первой колонке элемент 240px, а во второй — 760px, плагин вычислит пропорции и выдаст `1fr 3fr` вместо `auto auto`.

Если ты хочешь точные пропорции — задай их явно в Figma: `1fr 2fr` вместо `auto`.

### Именование слоёв

Имена слоёв конвертируются в CSS-классы:
- `Hero Section` → `.hero-section`
- `Card` → `.card`
- `image 32` → `.image-32`

Осмысленные имена → читаемый промпт → лучший код от AI. Не оставляй `Frame 427`.

### Spacing кратный 4px

Плагин проверяет, что padding и gap кратны 4px. Если нет — покажет warning. Tailwind-шкала основана на кратных 4, нестандартные значения превращаются в arbitrary (`gap-[13px]`), что хуже для поддержки.

### Text Styles и Color Variables

Плагин проверяет:
- Привязан ли текст к **Text Style** — если нет, покажет рекомендацию
- Привязан ли цвет к **Color Variable** — если нет, покажет рекомендацию

Это не блокирует работу, но помогает поддерживать макет чистым.

### Group vs Frame

`Group` не имеет layout-свойств — плагин выдаст warning. Замени на Frame с Auto Layout.

## Как читать результат

### Секция «Проблемы»

Маркеры `!` — вещи, которые стоит исправить в макете:
- **Нет Auto Layout** — добавь Shift+A
- **Отступы не кратны 4px** — округли
- **Group вместо Frame** — замени

Индикаторы: красная точка = ошибка, пустая точка = предупреждение.

### Секция «Позиционирование»

Дерево от корня до твоего элемента. Показывает, в каком flex/grid-контейнере он живёт. AI использует это, чтобы правильно вложить элемент в HTML-структуру.

### Секция «Характеристики»

Таблица свойств элемента с Tailwind-классами и raw-значениями. Цветные квадратики — preview цвета.

### Секция «Рекомендации»

Советы по улучшению макета (Text Style, Color Variable, нечётные размеры). Маркер `i` — не блокируют работу.

## Формат промпта

Скопированный текст выглядит так:

```
ОБЪЕКТ:
Section (FRAME)
Layout: CSS Grid: 2 cols × 2 rows; grid-template-columns: 1fr 3fr; grid-template-rows: auto auto; gap: 10px
Size: width: hug, height: hug
Overflow: clip content (overflow: hidden)
Fill: fill #ffffff
Tailwind: grid grid-cols-[1fr_3fr] gap-2.5 w-auto h-auto bg-white

ПОЗИЦИОНИРОВАНИЕ:
.page-wrapper (flex-col, gap-6, p-8)
  └─ Section [ЭТОТ ЭЛЕМЕНТ] (grid grid-cols-[1fr_3fr], gap-2.5, w-auto, h-auto)

ХАРАКТЕРИСТИКИ:
Фон: #ffffff → bg-white
Размер: 1010×610px

ВЛОЖЕННЫЕ ЭЛЕМЕНТЫ:
|- Card -- Auto Layout: vertical, align-items: center, justify: end; grid-area: 2 / 2; column track: 3fr; row track: auto; width: 760, height: 460; fill #f0f2f3; radius 24px [flex-col items-center justify-end bg-gray-100 rounded-3xl]
  |- image 32 -- width: 410, height: 553; fill #ff0000 [bg-[#ff0000]]
  |- Круг -- absolute; top: -40px; left: -40px (overflows parent); width: 40, height: 40; fill #d9d9d9; radius 9999px [bg-zinc-300 rounded-full]
|- Rectangle 36926 -- grid-area: 1 / 1; column track: 1fr; row track: auto; width: 240, height: 140; fill #d9d9d9; radius 0 0 1000 0 [bg-zinc-300 rounded-tl-none rounded-tr-none rounded-br-[1000px] rounded-bl-none]
|- [EMPTY CELL] -- grid-area: 1 / 2
```

**Что здесь важно:**
- CSS-свойства дословные — AI копирует их в код без додумывания
- Tailwind-классы в квадратных скобках `[...]` — готовы к вставке в `className`
- Grid-позиции с `grid-area` — AI ставит элемент точно в нужную ячейку
- `[EMPTY CELL]` — AI знает, что ячейка должна быть пустой
- Размеры треков (`column track: 3fr`) — AI понимает пропорции
- Absolute-позиции с CSS offsets — `top: -40px; left: -40px`

## Чеклист перед копированием промпта

- [ ] На всех контейнерах есть Auto Layout (или Grid)
- [ ] Нет warnings в секции «Проблемы»
- [ ] Слои осмысленно названы (не `Frame 427`)
- [ ] Spacing кратный 4px
- [ ] Если нужен контекст родителя — включён тоггл «Include parent context»

## Что плагин НЕ делает

- Не генерирует код — только описание для AI
- Не поддерживает множественное выделение — только один элемент
- Не работает с вариантами компонентов — анализирует конкретный instance
- Не читает прототипирование (interactions, transitions)

## Расширение плагина — что можно добавить

Если ты дизайнер и хочешь улучшить плагин:

### Новые свойства

В Figma есть свойства, которые пока не извлекаются:
- **Constraints** (pin to edges) — `node.constraints`
- **Component properties** — для variants

Для каждого нужно: извлечь значение из Figma API → замаппить на Tailwind-класс → добавить в промпт.

### Новые правила валидации

Можно добавить проверки:
- Слишком глубокая вложенность (>5 уровней)
- Текст без constrainted width (может вылезти за границы)
- Фиксированные размеры на адаптивных элементах
- Одинаковые цвета, не привязанные к одной переменной

### Улучшение промпта

Формат промпта можно дорабатывать:
- Добавить секцию «Responsive» с breakpoints
- Добавить секцию «Interactions» с Figma-прототипированием
- Генерировать промпт на английском (сейчас метки на русском)
- Добавить вариант промпта для конкретного фреймворка (React/Vue/Svelte)
